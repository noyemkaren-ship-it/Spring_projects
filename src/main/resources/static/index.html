🐅 Spring Boot — Полный Гайд от Тигра

Spring Boot — это фреймворк для создания веб-приложений на Java. Он позволяет поднять сервер за минуты и сосредоточиться на бизнес-логике, а не на настройке.

📦 Установка и первый запуск

1. Установи JDK 21 (или 17).

2. Установи VS Code и расширение Spring Boot Extension Pack.

3. Создай проект через Ctrl+Shift+P → Spring Initializr: Create a Gradle Project.

4. Добавь зависимости Spring Web, Spring Data JPA, H2 Database.

5. Запусти: gradle bootRun.

Модель (Entity) — аналог структуры в Go

@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    @Column(nullable = false)
    private int age;

    @Column(unique = true)
    private String email;

    // Пустой конструктор (обязателен для JPA!)
    public User() {}

    // Конструктор с параметрами
    public User(String name, int age, String email) {
        this.name = name;
        this.age = age;
        this.email = email;
    }

    // Геттеры и сеттеры (генерируются через Alt+Insert в IDEA)
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
}

Репозиторий — аналог GORM

Spring Data JPA позволяет создавать методы запросов просто по их имени. Тебе не нужно писать SQL!

public interface UserRepository extends JpaRepository<User, Long> {

    // Найти всех пользователей с таким именем
    List<User> findByName(String name);

    // Найти пользователя по email
    User findByEmail(String email);

    // Найти всех пользователей старше указанного возраста
    List<User> findByAgeGreaterThan(int age);

    // Найти по имени и возрасту
    List<User> findByNameAndAge(String name, int age);
}

Сервисный слой (правильная архитектура)

@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    public User getUserById(Long id) {
        return userRepository.findById(id)
                .orElseThrow(() -> new RuntimeException("Пользователь не найден"));
    }

    public User createUser(User user) {
        return userRepository.save(user);
    }

    public void deleteUser(Long id) {
        userRepository.deleteById(id);
    }
}

🌐 Эндпоинты (REST API)

Полный REST-контроллер с CRUD

@RestController
@RequestMapping("/api/v1/users")
public class UserController {

    @Autowired
    private UserService userService;

    // GET /api/v1/users — получить всех
    @GetMapping
    public ResponseEntity<List<User>> getAll() {
        return ResponseEntity.ok(userService.getAllUsers());
    }

    // GET /api/v1/users/1 — получить одного
    @GetMapping("/{id}")
    public ResponseEntity<User> getOne(@PathVariable Long id) {
        return ResponseEntity.ok(userService.getUserById(id));
    }

    // POST /api/v1/users — создать
    @PostMapping
    public ResponseEntity<User> create(@RequestBody User user) {
        return ResponseEntity.status(201).body(userService.createUser(user));
    }

    // PUT /api/v1/users/1 — обновить
    @PutMapping("/{id}")
    public ResponseEntity<User> update(@PathVariable Long id, @RequestBody User user) {
        user.setId(id);
        return ResponseEntity.ok(userService.createUser(user));
    }

    // DELETE /api/v1/users/1 — удалить
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        userService.deleteUser(id);
        return ResponseEntity.noContent().build();
    }

    // GET /api/v1/users/search?name=Тигр — поиск по параметру
    @GetMapping("/search")
    public ResponseEntity<List<User>> search(@RequestParam String name) {
        return ResponseEntity.ok(userRepository.findByName(name));
    }
}

🍪 Куки (Cookies)

Установка куки

@GetMapping("/set-cookie")
public String setCookie(HttpServletResponse response) {
    Cookie cookie = new Cookie("username", "Tiger");
    cookie.setMaxAge(3600);       // время жизни в секундах (1 час)
    cookie.setHttpOnly(true);     // JavaScript не может прочитать
    cookie.setSecure(true);       // только по HTTPS (для продакшена)
    cookie.setPath("/");          // доступна на всём сайте
    response.addCookie(cookie);
    return "Кука установлена!";
}

Чтение куки

// Способ 1: через аннотацию @CookieValue
@GetMapping("/get-cookie")
public String getCookie(@CookieValue(value = "username", defaultValue = "гость") String username) {
    return "Привет, " + username + "!";
}

// Способ 2: через HttpServletRequest (можно прочитать все куки)
@GetMapping("/all-cookies")
public String getAllCookies(HttpServletRequest request) {
    Cookie[] cookies = request.getCookies();
    StringBuilder sb = new StringBuilder();
    for (Cookie cookie : cookies) {
        sb.append(cookie.getName()).append(" = ").append(cookie.getValue()).append("; ");
    }
    return sb.toString();
}

Удаление куки

@GetMapping("/delete-cookie")
public String deleteCookie(HttpServletResponse response) {
    Cookie cookie = new Cookie("username", null);
    cookie.setMaxAge(0);          // 0 = удалить немедленно
    cookie.setPath("/");
    response.addCookie(cookie);
    return "Кука удалена!";
}

🔑 JWT Токены

build.gradle (зависимость)

implementation 'io.jsonwebtoken:jjwt-api:0.12.5'
runtimeOnly 'io.jsonwebtoken:jjwt-impl:0.12.5'
runtimeOnly 'io.jsonwebtoken:jjwt-jackson:0.12.5'

Утилитный класс для работы с JWT

@Component
public class JwtUtil {

    // Секретный ключ (в реальном проекте — в application.properties!)
    private final SecretKey secretKey = Keys.hmacShaKeyFor(
        "мой-супер-секретный-ключ-длиной-256-бит!!".getBytes()
    );

    // Генерация токена
    public String generateToken(String username) {
        return Jwts.builder()
                .setSubject(username)
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + 3600000)) // 1 час
                .signWith(secretKey)
                .compact();
    }

    // Извлечение имени пользователя из токена
    public String getUsernameFromToken(String token) {
        return Jwts.parser()
                .verifyWith(secretKey)
                .build()
                .parseSignedClaims(token)
                .getPayload()
                .getSubject();
    }

    // Проверка валидности токена
    public boolean isTokenValid(String token) {
        try {
            Jwts.parser().verifyWith(secretKey).build().parseSignedClaims(token);
            return true;
        } catch (Exception e) {
            return false;
        }
    }

    // Извлечение всех claims (данных) из токена
    public Claims getAllClaims(String token) {
        return Jwts.parser()
                .verifyWith(secretKey)
                .build()
                .parseSignedClaims(token)
                .getPayload();
    }
}

Middleware для проверки токена (аналог Gin Middleware)

@Component
public class JwtFilter extends OncePerRequestFilter {

    @Autowired
    private JwtUtil jwtUtil;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain chain)
            throws ServletException, IOException {

        String authHeader = request.getHeader("Authorization");

        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);

            if (jwtUtil.isTokenValid(token)) {
                String username = jwtUtil.getUsernameFromToken(token);
                // Здесь можно загрузить пользователя из БД и положить в SecurityContext
                System.out.println("Авторизован: " + username);
            }
        }

        chain.doFilter(request, response);
    }
}

📄 Как возвращать HTML

Способ 1: Статические файлы (самый простой)

Просто положи файл в src/main/resources/static/. Spring Boot автоматически отдаст его по прямому пути.

src/main/resources/static/index.html → http://localhost:8080/index.html
src/main/resources/static/css/style.css → http://localhost:8080/css/style.css
src/main/resources/static/js/script.js → http://localhost:8080/js/script.js

Способ 2: @Controller (возвращает имя файла)

@Controller  // ← НЕ @RestController!
public class HomeController {

    @GetMapping("/")
    public String home() {
        return "index.html";  // ищет в static/
    }

    @GetMapping("/about")
    public String about() {
        return "about.html";
    }
}

Способ 3: Редирект (через @RestController)

@RestController
public class HomeController {

    @GetMapping("/home")
    public String home() {
        return "redirect:/index.html";
    }
}

Способ 4: ModelAndView с шаблонизатором Thymeleaf

// Добавь в build.gradle:
// implementation 'org.springframework.boot:spring-boot-starter-thymeleaf'

@Controller
public class HomeController {

    @GetMapping("/greet")
    public ModelAndView greet() {
        ModelAndView mav = new ModelAndView("greet"); // greet.html в templates/
        mav.addObject("name", "Тигр");
        mav.addObject("age", 14);
        return mav;
    }
}
<!-- templates/greet.html -->
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<body>
    <h1>Привет, <span th:text="${name}"></span>!</h1>
    <p>Тебе <span th:text="${age}"></span> лет.</p>
</body>
</html>
